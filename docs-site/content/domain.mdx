# Domain

คำศัพท์หน้านี้ใช้เป็นภาษากลางระหว่าง Product, Engineering และ Operations ชื่อภาษาอังกฤษที่กำหนดไว้ควรถูกใช้ตรงกันใน UI, audit trail และเอกสาร เพื่อไม่ให้ state เดียวกันถูกตีความเป็นคนละความหมาย

## Identity and Configuration

| คำศัพท์ | ความหมาย |
| --- | --- |
| **Binance Account** | บัญชีเดียวที่ installation ใช้งาน Credentials สำหรับ Live เก็บใน OS Keyring และไม่ถูกบันทึกใน application database |
| **Bot Session** | การทำงานหนึ่งรอบที่ตรึง Market Type, Mode, Preset version, Available Capital, Entry Policy, market-specific capital policy และ execution costs ไว้ด้วยกัน หนึ่ง installation มี Active Bot Session ได้สูงสุดหนึ่ง Session |
| **Preset** | ชุดพารามิเตอร์ Strategy แบบ immutable และมี version การแก้ค่าต้องสร้าง version ใหม่โดยไม่เปลี่ยน Session ที่เริ่มไปแล้ว |

Bot Session เป็น ownership boundary โดยตรงของ Basket, Entry, Order, Fill, PnL และ audit records ทุก trading record ต้องย้อนกลับไปยัง `session_id` ที่สร้างรายการนั้นได้

## Market and Decision

| คำศัพท์ | ความหมาย |
| --- | --- |
| **Candle** | ข้อมูล OHLC และ volume ของ BTCUSDT ช่วง 5 นาทีที่เปิดตรงขอบเวลา UTC Strategy รับเฉพาะ completed Candle |
| **Entry Intent** | ผลการตัดสินใจแบบ deterministic ว่าจะเปิด Entry ลำดับใดจาก signal Candle และ ATR ใด พร้อม idempotency key ก่อน execution |
| **Entry** | ส่วนหนึ่งของ Position ที่ Bot เปิดจาก Entry Intent และได้รับการยืนยันด้วย Fill หนึ่ง Basket มีได้สูงสุดตาม `max_entries` ของ Session |
| **Fill** | ข้อเท็จจริงว่า Order ถูกจับคู่แล้ว พร้อมราคา quantity, fee และเวลาที่ใช้ปรับ state |

Entry Intent ยังไม่ใช่ Entry ที่เกิดขึ้นจริง การเปลี่ยน state ต้องอาศัย Fill เพื่อป้องกันการนับ Position จากเพียงความพยายามส่ง Order

## Basket and Lifecycle

| คำศัพท์ | ความหมาย |
| --- | --- |
| **Basket** | กลุ่ม Entries ของรอบการสะสม Position เดียวกัน มี weighted average, quantity รวม และ Take Profit ร่วมกัน |
| **Take Profit** | Limit สำหรับปิด Position ที่ Bot เป็นเจ้าของทั้งหมดใน Basket เมื่อ Fill แล้ว Entries ทั้งหมดปิดพร้อมกัน |
| **Entry Pair** | การจัด Entries เป็นคู่ตามลำดับ `(2n - 1)` และ `2n` จนครบ `max_entries` เพื่อควบคุมจังหวะเพิ่ม Position |
| **Cooldown Month** | เดือน UTC หลัง Entry Pair ครบและ Basket ยังไม่ปิด ซึ่งห้ามเพิ่ม Entry แต่ Take Profit ยังทำงาน |

Entry เดี่ยวของ Pair สามารถรอ Entry ที่สองข้ามเดือนได้ เมื่อ Pair ครบแล้วจึงเริ่มกฎ Cooldown Month การปิด Basket จะ reset Entry Pair และ Cooldown state ทำให้ Basket ใหม่เริ่มได้ทันทีแม้อยู่ในเดือน UTC เดิม

## Capital and Execution

| คำศัพท์ | ความหมาย |
| --- | --- |
| **Trading Capital** | ส่วนของ Available Capital ที่ใช้คำนวณขนาด Entries ตาม immutable Session policy |
| **Reserve** | ส่วนที่เหลือจาก Spot Trading Capital Ratio ซึ่งไม่ใช้คำนวณขนาด Entry; ค่าเริ่มต้นเท่ากับ 20% |
| **Collateral Buffer** | 50% ของ Futures Available Capital ที่ไม่ใช้สร้าง Entry ใหม่ แต่เป็นเงินที่ผู้ใช้ยอมเสี่ยงเพื่อเลื่อน liquidation |
| **Paper** | Mode ที่ใช้ conservative candle fill และข้อมูลที่ตรึงไว้เพื่อ replay ผลลัพธ์เดิมโดยไม่ส่งคำสั่งจริง |
| **Live** | Mode ที่ส่งคำสั่งผ่าน adapter เฉพาะ Spot หรือ Futures หลัง Preflight และการยืนยันจากผู้ใช้ |

Business policies เหมือนกันระหว่าง Paper และ Live ความต่างอยู่ที่ execution adapter และข้อกำหนด safety รอบการเชื่อมต่อ exchange

## Recovery and Reconciliation

**Recovery** คือกระบวนการเปิดโปรแกรมแล้วโหลด Bot Session, Basket, Entry, Order และ Fill จาก SQLite ก่อนตัดสินใจว่าจะ Resume ได้หรือไม่

**Reconciliation** คือการเปรียบเทียบ local state กับ exchange state สำหรับ Live Session หากข้อมูลไม่ตรงกัน ระบบบล็อก Entry ใหม่และรอให้ผู้ใช้แก้ไข โดยไม่ยกเลิก Order หรือ rewrite state อัตโนมัติ

Recovery จัดการการกลับมาจาก state ที่บันทึกไว้ ส่วน Reconciliation พิสูจน์ว่า state นั้นสอดคล้องกับ exchange ทั้งสองคำจึงสัมพันธ์กันแต่ไม่ใช้แทนกัน
