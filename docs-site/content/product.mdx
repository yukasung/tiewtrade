# Product

TiewTrade เป็น local single-user desktop application สำหรับเจ้าของผลิตภัณฑ์และผู้ทดสอบภายในกลุ่มเล็ก macOS คือแพลตฟอร์มแรก แต่ขอบเขตสถาปัตยกรรมไม่ปิดทาง Windows และ Linux ในอนาคต เป้าหมายสำคัญกว่าจำนวนฟีเจอร์คือผลลัพธ์ที่ตรวจสอบย้อนหลังได้ กู้คืนได้ และไม่สร้าง Order ซ้ำ

## Internal Alpha Scope

| มิติ | ขอบเขตที่รองรับ |
| --- | --- |
| Platform | macOS เป็นแพลตฟอร์มแรก |
| Symbol และข้อมูล | BTCUSDT, completed Candle 5 นาที, UTC |
| Strategy | RSI Step Grid จาก Versioned Preset หนึ่งกลยุทธ์ |
| Market และ Mode | Paper Spot, Paper Futures, Live Spot และ Live Futures ตาม Delivery Gates |
| Account | Binance Sub-account สูงสุด 5 Account Profiles และหนึ่ง Active Bot Session ต่อ Profile |
| Application | Local single-user desktop application, UI ภาษาอังกฤษ |
| Operations | Persistence, audit trail, Stop Session, Recovery และ Reconciliation |

ระบบรองรับ BTCUSDT เพียง Symbol เดียว, completed candle 5 นาที, เวลา UTC และ RSI Step Grid จาก Versioned Preset หนึ่งกลยุทธ์ รองรับ Paper Spot, Paper USDⓈ-M Futures, Live Spot และ Live USDⓈ-M Futures ตาม Delivery Gates ที่กำหนด

ผู้ใช้สร้าง Account Profiles จาก Binance Sub-account ที่มีอยู่แล้วได้สูงสุด 5 รายการ แต่ละ Account Profile มี credentials แยกกันและมี Active Bot Session ได้สูงสุดหนึ่ง Session UI เป็นภาษาอังกฤษและต้องแยก Paper กับ Live ให้เห็นชัด

ขอบเขตนี้รวม market data, Strategy, Basket, Order และ Fill, SQLite persistence, Operations UI, Stop Session, Recovery และ Reconciliation เพราะแต่ละ milestone ต้องเป็น vertical slice ที่ใช้งานและตรวจสอบได้ครบเส้นทาง

อ่าน [Domain Reference](/domain) เพื่อใช้คำศัพท์และกฎโดเมนชุดเดียวกันตลอดทั้งระบบ

## Delivery Gates

แผนภาพต่อไปนี้แสดง gate แบบสะสมที่ใช้ควบคุมการเปิด execution mode ทีละระดับ

```mermaid
flowchart LR
    A[Paper Trading Complete] -->|quality gate| B[Live Spot]
    B -->|quality gate| C[Live Futures]
```

ลำดับนี้เป็น gate แบบสะสม: Live Spot เริ่มได้เมื่อ Paper ครบทุก acceptance criterion และ Live Futures เริ่มได้เมื่อ Live Spot ผ่านแล้วเท่านั้น ทั้งสองขั้น Live ยังต้องได้รับการอนุมัติอย่างชัดแจ้งจากผู้ใช้

Paper Trading Complete ต้องพิสูจน์ completed Candle ที่ต่อเนื่อง, Strategy และ lifecycle ที่ถูกต้อง, deterministic replay, fee/slippage/funding, Recovery ที่ไม่สร้าง Entry หรือ Order ซ้ำ, การแยก Account Profile และ UI ที่แสดง state สำคัญ

Live Spot เพิ่ม Preflight, idempotency, Reconciliation และ safe shutdown โดยทดสอบกับ fake adapter ก่อน ส่วน Live Futures เพิ่ม Cross Margin, leverage cap, Collateral Buffer, funding และข้อเท็จจริงที่เกี่ยวกับ liquidation

## Risk Policy

Internal Alpha แสดง PnL และ Drawdown เพื่อเฝ้าระวัง แต่ยังไม่มี Stop Loss, Maximum Drawdown hard limit, Max Daily Loss หรือ Consecutive-loss stop ระบบจึงไม่บังคับปิด Position จากตัวเลขเหล่านี้ อย่างไรก็ตาม capital allocation, maximum 10 Entries และ Futures leverage ไม่เกิน 5x เป็นข้อบังคับเสมอ

## Out of Scope

| มิติ | สิ่งที่ไม่รองรับในรุ่นนี้ |
| --- | --- |
| Market และ Strategy | Symbol อื่น, หลายกลยุทธ์ และ Strategy Plugin System |
| Exchange และ Trading | Master Account API, Binance Testnet และ manual order terminal |
| Analysis และ Risk | advanced chart tools, Backtest UI และ automatic loss limits |
| Identity และ Distribution | Login, ระบบสมาชิก และ License |
| Platform และ Sync | Cloud Sync, Web application และ Mobile application |

การเปลี่ยน business rule, risk policy หรือ Live Gate ต้องเกิดขึ้นอย่างเปิดเผย และเมื่อกระทบ Strategy Session ต้องสร้าง Preset version ใหม่โดยไม่เปลี่ยนพฤติกรรมของ Session เดิมย้อนหลัง
