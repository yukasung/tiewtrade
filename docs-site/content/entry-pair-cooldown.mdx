# Entry Pair & Cooldown

Entries ใน Basket ถูกจัดเป็นคู่เพื่อเว้นจังหวะการเพิ่ม Position ตามเดือน UTC กฎนี้จำกัดการสะสมโดยไม่หยุด Take Profit จำนวน Entries สูงสุดมาจาก immutable Session policy และต้องเป็นเลขคู่ 2–20

## Pair Sequence

ตัวอย่างนี้ใช้ค่าเริ่มต้น 10 Entries จึงมี Pair ทั้งหมดห้าคู่

```mermaid
flowchart LR
    P1[Pair 1<br/>Entries 1-2] --> P2[Pair 2<br/>Entries 3-4]
    P2 --> P3[Pair 3<br/>Entries 5-6]
    P3 --> P4[Pair 4<br/>Entries 7-8]
    P4 --> P5[Pair 5<br/>Entries 9-10]
```

แผนภาพแสดง Pair ทั้งห้าตามลำดับ ระบบเริ่ม Pair ถัดไปได้เมื่อ Pair ปัจจุบันครบสอง Entries, ผ่าน Cooldown Month ที่เกี่ยวข้อง และ Basket ยังเปิดอยู่

แต่ละ Entry ยังต้องมี reset และ confirmation ของ Strategy ใหม่ การอนุญาตจาก lifecycle ไม่ได้สร้าง Entry เอง และ `max_entries` ของ Session ยังคงเป็นเพดานสุดท้าย

## UTC Month Boundary

State diagram นี้แสดงเงื่อนไขเปลี่ยนจาก Entry เดี่ยวไปยัง Pair ที่ครบ, Cooldown Month และ Pair ถัดไป

```mermaid
stateDiagram-v2
    [*] --> PairOpen
    PairOpen --> SingleEntry: first Entry Fill
    SingleEntry --> SingleEntry: processed completed Candle enters a new UTC month
    SingleEntry --> PairComplete: second Entry Fill
    PairComplete --> CooldownMonth: processed Candle month follows Pair completion month
    CooldownMonth --> NextPair: processed Candle month follows Cooldown Month
    NextPair --> PairOpen
    PairOpen --> [*]: Basket closes
    SingleEntry --> [*]: Basket closes
    PairComplete --> [*]: Basket closes
    CooldownMonth --> [*]: Basket closes
```

แผนภาพแยกข้อยกเว้นสำคัญโดยใช้เดือน UTC ของ completed Candle ที่กำลังประมวลผล ไม่ใช่เวลานาฬิกาของเครื่อง: Entry เดี่ยวสามารถรอ Entry คู่ของตนข้ามสิ้นเดือน UTC ได้ แต่เมื่อ Pair ครบและ Basket ยังไม่ปิด completed Candles ของเดือน UTC ถัดไปอยู่ใน Cooldown Month ก่อนเปิด Pair ถัดไป

ตัวอย่าง: Entry 1 Fill วันที่ 31 มกราคม และ Entry 2 Fill วันที่ 2 กุมภาพันธ์ ถือว่า Pair 1 ครบในเดือนกุมภาพันธ์ เดือนมีนาคมเป็น Cooldown Month และ Pair 2 เริ่มได้ตั้งแต่เดือนเมษายน หาก Basket ยังไม่ปิด

Timeline ต่อไปนี้ทำให้ตัวอย่างเดียวกันเห็นลำดับเดือน UTC และจุดที่ Pair ถัดไปกลับมามีสิทธิ์

```mermaid
flowchart LR
    JAN["มกราคม UTC<br/>Entry 1"] --> FEB["กุมภาพันธ์ UTC<br/>Entry 2 completes Pair 1"]
    FEB --> MAR["มีนาคม UTC<br/>Cooldown Month"]
    MAR --> APR["เมษายน UTC<br/>Pair 2 eligible"]
```

แผนภาพ timeline ยืนยันว่า Entry เดี่ยวข้ามเดือนได้ และ Cooldown เริ่มจาก completed Candles ของเดือนหลัง Pair ครบ ไม่ได้เริ่มจากเวลานาฬิกาของเครื่อง

## During the Cooldown Month

ระบบห้ามสร้าง Entry ใหม่ แต่ยังติดตาม market data, อัปเดต UI และรักษา Basket Take Profit ตามปกติ Cooldown ไม่ยกเลิก Order, ไม่ปิด Position และไม่หยุด Recovery

เมื่อ Basket ปิดเมื่อใด lifecycle ของ Basket นั้นสิ้นสุดทันที Pair และ Cooldown state ถูก reset ทำให้ Basket ใหม่เริ่ม Pair 1 ในเดือน UTC เดียวกันได้ ไม่ต้องรอ Cooldown เดิม

## Entry Eligibility Evaluation

ก่อนอนุมัติ Entry Intent ระบบตรวจตามลำดับว่า Bot Session รับ Entry ใหม่ได้, Basket ยังไม่ครบ `max_entries`, ไม่มี Intent รอ Fill, Pair ปัจจุบันรับ Entry ได้ และเดือน UTC ของ completed Candle ที่กำลังประมวลผลไม่ใช่ Cooldown Month หลังจากนั้นจึงคำนวณขนาดจาก [Capital Allocation](/capital-allocation)
